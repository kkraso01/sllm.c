# RUN_LOCAL.md

Primary guide for running the ALC-local reproducibility path on a normal laptop.

---

## A) Requirements

### OS assumptions
- Linux or macOS (bash shell).
- Commands below assume repo root as current working directory.

### Compiler/runtime requirements
- C compiler (`cc`, `clang`, or `gcc`).
- OpenMP runtime (`-fopenmp`, `-lgomp` on Linux).
- `make`.
- Python 3.8+.

### Optional tools
- `matplotlib` for figure generation in paper artifacts.
- LaTeX (`pdflatex`, `bibtex`) for PDF paper build.

### What is *not* required for this local ALC pass
- No GPU is required.
- No CUDA/nvcc is required.
- No large pretrained model artifacts are required for the publication-style ALC benchmarks.

---

## B) Quick start

### 1) Build local binaries/tests
```bash
./scripts/local/build_all.sh
```

### 2) Run important artifact-independent tests
```bash
./scripts/local/run_tests.sh
```

### 3) Run tiny local ALC demo (first run recommended)
```bash
./scripts/local/run_alc_demo.sh
```
This writes metrics and state files under `artifacts/local_demo/`.

### 4) Run publication suite (paper artifacts path)
```bash
./scripts/run_alc_publication_suite.sh
```
This writes outputs under `paper/results/`, `paper/tables/`, and optionally `paper/figures/`.

## C) Baseline vs `alc_no_write` vs `alc`

### Primary benchmark (SessionKV-DR) one-command local path
```bash
./scripts/local/run_sessionkv.sh
```

This runs the benchmark (seed 42), then prints and stores the key SessionKV-DR comparison:
- `baseline`
- `alc_no_write`
- `alc`

Key metric shown by default:
- `delayed_acc_delay_6_facts_4`

Outputs:
- `artifacts/sessionkv/metrics_sessionkv_seed42.csv`
- `artifacts/sessionkv/sessionkv_summary_seed42.json`

## D) Persistence demo (save/load ALC state)

Run the scripted save/load demo:
```bash
./scripts/local/run_persistence_demo.sh
```

What this demonstrates:
1. **Run A** writes SessionKV state sidecars (`baseline`, `alc_no_write`, `alc`).
2. **Run B** reads metrics proving save/load correctness and retained recall behavior.

Look for these metrics in the script output:
- `persistence_save_ok`
- `persistence_load_ok`
- `persistence_recall_acc`
- (ALC identity checks) `persistence_slot_max_abs_diff`, `persistence_behavior_max_abs_diff`

## F) Common failures and troubleshooting

### 1) OpenMP/linker failures
Symptoms:
- compile error for `-fopenmp` or link error for `-lgomp`.

Fixes:
- Ubuntu/Debian: install build tools + OpenMP runtime (`libomp-dev` and/or `libgomp1` depending on compiler).
- macOS (clang): install `libomp` via Homebrew and use clang OpenMP flags.
- As a fallback, update scripts to remove OpenMP flags for very small demo runs (slower, and not the default path).

### 2) Missing artifacts/directories
Symptoms:
- script says output path/file missing.

Fixes:
- Run from repo root.
- Re-run `./scripts/local/build_all.sh`.
- Confirm script-created output directories exist (`artifacts/...` or `paper/results/...`).

### 3) Plot generation unavailable
Symptoms:
- figures not generated by `scripts/make_paper_artifacts.py`.

Expected behavior:
- a warning file is written to `paper/figures/FIGURE_GENERATION_WARNING.txt`.

Fix:
- Install matplotlib if you need plots.

### 4) LaTeX not installed
Symptoms:
- `pdflatex`/`bibtex` command not found when building paper PDF.

Fix:
- Install TeX distribution (optional; not required for benchmark reproducibility).

### 5) Path mistakes
Symptoms:
- `No such file or directory` for scripts or binaries.

Fix:
- Ensure you are at repository root and run scripts as `./scripts/local/<script>.sh`.
