\begin{abstract}
This work studies ALC, an architecture-level mechanism for runtime memory updates during inference while keeping backbone parameters fixed. We evaluate ALC with SessionKV-DR, a controlled benchmark that isolates delayed recall, overwrite correctness, persistence across save/reload, and interference effects. Using baseline, no-write, and full ALC variants with multi-seed reporting, we find mechanism-level gains in delayed session-specific recall and stable state behavior in the tested configuration. We also report where no-write remains competitive, indicating unresolved write-policy regimes. TinyQA is presented as supporting evidence only. The paper emphasizes claims discipline and clear boundaries on what the current results do and do not establish.
\end{abstract}
